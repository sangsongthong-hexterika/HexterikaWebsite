Every AWS account starts with Identity & Access Management (IAM). Done right, IAM is the foundation that keeps your team secure. Done wrong, it's the weak point that hackers, misconfigurations, and billing confusion exploit.

I follow AWS IAM best practices to build a clean foundation. That means:

  • MFA enforced for all users.
  • Roles and groups that follow least-privilege principles.
  • Billing separation ‚Äî only finance (and authorized leadership) get billing rights, so charges don't spiral out of control or get exposed to the wrong hands.
  • Simple guide so you can safely add new users later without creating a mess.

Includes:

  • IAM structure (users, groups, roles) using best-practice AWS-managed policies
  • MFA required for all users, billing access isolated
  • Password and access key policies configured

Deliverable:

  • User-friendly guide with IAM overview and step-by-step instructions to add new users safely

Scope:

  • One AWS account
  • Up to 10 users
  • For new or clean accounts only (no active workloads or custom IAM policies)

Not included:

  • Multi-account AWS Organization setup
  • SSO or federated access configuration
  • Custom service-specific IAM policies (available as add-on at $10 per policy)

Client needs to provide:

  • Temporary admin access or screenshare session
  • List of users with names, roles, and what services they need access to (e.g., EC2, Billing, S3)
